## 🚨 발견된 P0 블로커 버그 (2026-04-03 업데이트)

### 버그: 장르 변경 시 스토리가 새로 생성되지 않음

**버그 ID:** AI-252-BUG-001
**발견 시점:** 2번째 장르(무협) 테스트 진행 중
**우선순위:** P0 (Critical)

#### 증상
- 스토리 에디터에서 장르 프리셋을 변경하여 새로운 스토리를 생성하고 저장
- "게임 시작" 버튼 클릭 시 이전 세션의 스토리가 로드됨
- 시스템 프롬프트는 새 장르로 변경되었으나, 실제 게임 콘텐츠는 이전 장르 유지
- **부정합 발생:** 무협 프롬프트 vs 판타지 스토리 콘텐츠

#### 재현 단계
1. 기존 스토리(판타지 F1-Basic)에서 에디터 진입
2. 장르 변경 (판타지 → 무협)
3. 프리셋 변경 (🐉 에테르나의 전설 → ⚔️ 고전 무협: 마종의 제자)
4. 제목 변경 (F1-Basic → M1-Basic)
5. "저장" 버튼 클릭
6. "게임 시작" 버튼 클릭
7. **결과:** 판타지 스토리가 다시 로드됨

#### 영향
- ✗ AI-252 태스크 차단: 9개 장르별 품질 평가 불가능
- ✗ 프로덕션 기능 장애: 사용자가 새 장르의 스토리를 생성할 수 없음
- ✗ 사용자 경험 저조: 같은 스토리만 반복 생성됨

#### 기술적 원인 (추정)
1. **세션 ID 재사용:** 게임 시작 시 새 세션을 생성하지 않고 이전 세션 재사용
2. **프론트엔드 상태 관리 오류:** 에디터 변경 사항이 게임 세션에 반영되지 않음
3. **백엔드 API 라우팅 오류:** POST /api/game/start에서 스토리 설정 제대로 전달 안 됨

#### 수정 권장사항

**즉시 조치 (Hotfix):**
- 게임 시작 시 세션 무조건 새로 생성
- 스토리 변경 시 이전 세션 삭제

**근본적 조치:**
- POST /api/game/start에서 스토리 ID 필수화
- 프론트엔드 상태 관리 개선 (스토리 변경 시 세션 초기화)
- 장르 변경 E2E 테스트 케이스 추가
