## 3. 심각한 결함 (Critical Failures)

### 3.1 E2E 테스트 인프라 전체 실패

**결함 ID:** SYS-BUG-001
**심각도:** 🔴 CRITICAL
**영향 범위:** 모든 E2E 테스트 (34개)

**재현 단계:**
1. `npx playwright test` 실행
2. 모든 테스트가 `page.goto()`에서 실패

**예상 동작:**
- baseURL(`http://localhost:5173`) 기반으로 상대 경로 탐색 성공
- 테스트가 정상 실행

**실제 동작:**
```
Error: page.goto: Protocol error (Page.navigate): Cannot navigate to invalid URL
Call log:
  - navigating to "/", waiting until "load"
```

**근본 원인 (분석):**

**가능한 원인:**
1. **Page Object Model 구현 오류:** `e2e/pages/home.page.ts`와 `e2e/pages/play.page.ts`에서 `page.goto('/')` 호출 시 Playwright가 baseURL을 제대로 합성하지 못함
2. **Playwright Config baseURL 설정 문제:** `playwright.config.ts`에 baseURL이 설정되어 있어도 페이지 객체 생성 시 전달되지 않았을 가능성
3. **Browser Context 초기화 문제:** global-setup.ts에서 브라우저 컨텍스트가 baseURL 없이 초기화됨

**영향:**
- 모든 E2E 테스트 실행 불가
- 시스템 테스트 자동화 100% 차단
- 회귀 테스트 불가

**권장 조치:**
1. **긴급 (즉시):** 페이지 객체에서 `page.goto('/')` 대신 `page.goto('http://localhost:5173/')` 사용 (임시 완화)
2. **단기 (1시간 이내):** baseURL 설정 점검 및 페이지 객체 모델 수정
3. **중기 (1일 이내):** E2E 테스트 실행 방법 문서화 (`frontend/package.json`에 `test:e2e` 스크립트 추가)

**담당자:** Fullstack Dev 또는 QA Engineer
**예상 수정 시간:** 1-2시간
