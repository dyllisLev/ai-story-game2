## 1. 개요 (Overview)

### 1.1 시스템 테스트 개요

시스템 테스트는 SRS(요구사항 정의서)에 정의된 43개 기능 요구사항과 비기능 요구사항에 대해 **전체 시스템 수준에서 검증**하는 테스트 단계입니다.

### 1.2 테스트 범위

- **전체 유저 플로우:** 회원가입 → 스토리 탐색 → 게임 플레이 → 세션 관리
- **비기능 요구사항:** 성능, 보안, 접근성
- **크로스 브라우저:** Chrome (프로덕션 타겟)
- **RLS 정책:** Supabase Row Level Security 전체 검증

### 1.3 테스트 전제 조건

- **서비스 상태:** Backend (port 3000), Frontend (port 5173) 실행 중
- **테스트 데이터:** Supabase에 초기 스토리 데이터 존재
- **테스트 계정:**
  - 일반 사용자: `test@example.com` / `Test1234!`
  - 관리자: `admin@example.com` (role=admin)
