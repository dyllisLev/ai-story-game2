## UC-001: 스토리 탐색 및 선택

### 개요
| 항목 | 내용 |
|------|------|
| **유스케이스명** | 스토리 탐색 및 선택 |
| **ID** | UC-001 |
| **주요 액터** | 비로그인 사용자 |
| **설명** | 사용자가 공개 스토리 목록을 탐색하고, 원하는 스토리를 선택하여 상세 정보를 확인한다 |

### 선행 조건
- 애플리케이션이 실행 중이어야 한다
- 인터넷 연결이 되어 있어야 한다

### 메인 흐름 (Main Flow)
1. 사용자가 홈 페이지(`/`)에 접근한다
2. 시스템은 추천 스토리(Featured)를 히어로 섹션에 표시한다
3. 시스템은 전체 공개 스토리 목록을 표시한다 (기본 20개)
4. 사용자는 다음 필터를 적용할 수 있다:
   - **장르 필터**: 무협, 판타지, 로맨스 등 태그 선택
   - **검색**: 제목/설명 키워드 입력
   - **정렬**: 최신순, 인기순, 이름순
5. 시스템은 필터링된 결과를 실시간으로 업데이트한다
6. 사용자가 스토리 카드를 클릭한다
7. 시스템은 스토리 상세 정보를 표시한다:
   - 제목, 설명, 배너 이미지
   - 세계관, 캐릭터, 시스템 규칙 (일부 표시)
   - 플레이 수, 좋아요 수

### 대안 흐름 (Alternative Flows)
- **4a. 페이지네이션:**
  - 사용자가 "더보기"를 클릭한다
  - 시스템은 다음 페이지 스토리를 로드한다
- **6a. 로그인 상태:**
  - 사용자가 로그인 상태이면 "이어하기" 섹션에 최근 세션을 표시한다

### 사후 조건
- 사용자가 원하는 스토리를 선택하고 플레이를 시작할 수 있다

### 연관 요구사항
- STORY-BROWSE-001 ~ STORY-BROWSE-006

## UC-002: 회원가입

### 개요
| 항목 | 내용 |
|------|------|
| **유스케이스명** | 회원가입 |
| **ID** | UC-002 |
| **주요 액터** | 방문자 |
| **설명** | 새로운 사용자가 이메일과 비밀번호로 계정을 생성한다 |

### 선행 조건
- 유효한 이메일 주소를 가지고 있어야 한다

### 메인 흐름 (Main Flow)
1. 사용자가 회원가입 페이지(`/signup`)에 접근한다
2. 시스템은 회원가입 폼을 표시한다:
   - 이메일
   - 비밀번호 (8자 이상)
   - 비밀번호 확인
3. 사용자가 정보를 입력하고 "가입하기"를 클릭한다
4. 시스템은 입력값을 유효성 검사한다:
   - 이메일 형식 확인
   - 비밀번호 일치 확인
   - 비밀번호 강도 확인
5. 시스템은 Supabase Auth로 회원가입 요청을 전송한다
6. Supabase는 `auth.users` 테이블에 사용자를 생성한다
7. 트리거가 `user_profiles` 테이블에 자동으로 프로필을 생성한다
8. 시스템은 사용자를 로그인 페이지로 리다이렉트한다

### 예외 흐름 (Exception Flows)
- **4a. 유효성 검사 실패:**
  - 시스템은 오류 메시지를 표시하고 입력을 다시 요청한다
- **5a. 이메일 중복:**
  - Supabase가 "이미 존재하는 이메일" 오류를 반환한다
  - 시스템은 "이미 가입된 이메일입니다" 메시지를 표시한다

### 사후 조건
- 사용자 계정이 생성되고 로그인할 수 있다

### 연관 요구사항
- AUTH-001

## UC-003: 로그인

### 개요
| 항목 | 내용 |
|------|------|
| **유스케이스명** | 로그인 |
| **ID** | UC-003 |
| **주요 액터** | 방문자 |
| **설명** | 사용자가 이메일과 비밀번호로 인증하여 세션을 시작한다 |

### 선행 조건
- 사용자 계정이 이미 생성되어 있어야 한다

### 메인 흐름 (Main Flow)
1. 사용자가 로그인 페이지(`/login`)에 접근한다
2. 시스템은 로그인 폼을 표시한다:
   - 이메일
   - 비밀번호
3. 사용자가 정보를 입력하고 "로그인"을 클릭한다
4. 시스템은 Supabase Auth로 인증 요청을 전송한다
5. Supabase가 access token과 refresh token을 반환한다
6. 시스템은 토큰을 localStorage에 저장한다
7. 시스템은 사용자를 홈 페이지로 리다이렉트한다
8. 시스템은 헤더에 사용자 닉네임과 "로그아웃" 버튼을 표시한다

### 예외 흐름 (Exception Flows)
- **4a. 인증 실패:**
  - Supabase가 "잘못된 자격증명" 오류를 반환한다
  - 시스템은 "이메일 또는 비밀번호가 올바르지 않습니다" 메시지를 표시한다

### 사후 조건
- 사용자가 인증되고 모든 기능에 접근할 수 있다

### 연관 요구사항
- AUTH-002, AUTH-006

## UC-004: API 키 등록

### 개요
| 항목 | 내용 |
|------|------|
| **유스케이스명** | API 키 등록 |
| **ID** | UC-004 |
| **주요 액터** | 로그인 사용자 |
| **설명** | 사용자가 자신의 Gemini API 키를 안전하게 등록한다 |

### 선행 조건
- 사용자가 로그인 상태여야 한다
- 유효한 Google Gemini API 키를 가지고 있어야 한다

### 메인 흐름 (Main Flow)
1. 사용자가 설정 페이지(`/settings`)에 접근한다
2. 사용자가 "API 키 관리" 섹션을 찾는다
3. 사용자가 API 키를 입력하고 "저장"을 클릭한다
4. 시스템은 API 키를 AES-256으로 암호화한다
5. 시스템은 암호화된 키를 `user_profiles.api_key_encrypted`에 저장한다
6. 시스템은 "API 키가 저장되었습니다" 메시지를 표시한다

### 예외 흐름 (Exception Flows)
- **4a. API 키 유효성 검사:**
  - 시스템은 테스트 요청을 보내 API 키 유효성을 확인할 수 있다 (선택)
  - 유효하지 않으면 "유효하지 않은 API 키입니다" 메시지를 표시한다

### 사후 조건
- 사용자가 게임 플레이 시 등록한 API 키를 사용할 수 있다

### 연관 요구사항
- AUTH-005, NFR-SEC-001

## UC-005: 새 게임 세션 시작

### 개요
| 항목 | 내용 |
|------|------|
| **유스케이스명** | 새 게임 세션 시작 |
| **ID** | UC-005 |
| **주요 액터** | 로그인 사용자 |
| **설명** | 사용자가 스토리를 선택하고 새 게임 세션을 시작한다 |

### 선행 조건
- 사용자가 로그인 상태여야 한다
- API 키가 등록되어 있어야 한다
- 스토리가 선택되어 있어야 한다

### 메인 흐름 (Main Flow)
1. 사용자가 스토리 상세 페이지에서 "플레이"를 클릭한다
2. 시스템은 플레이 페이지(`/play/:storyId`)로 이동한다
3. 사용자가 "새 세션 시작"을 클릭한다
4. 시스템은 다음을 수행한다:
   - `stories` 테이블에서 스토리를 조회한다
   - 시스템 프롬프트를 조합한다:
     - world_setting
     - story
     - characters
     - system_rules
     - 상태창 규칙
   - `sessions` 테이블에 새 행을 생성한다
5. 시스템은 다음을 반환한다:
   - sessionId
   - sessionToken
   - systemPrompt
   - startMessage
   - safetySettings
6. 프론트엔드는 `useGameEngine` 상태를 초기화한다
7. 시스템은 시작 메시지를 표시한다

### 대안 흐름 (Alternative Flows)
- **4a. 상태창 프리셋이 있는 경우:**
  - 시스템은 `status_presets` 테이블에서 속성 정의를 조회한다
  - 시스템은 상태창 초기값을 설정한다

### 사후 조건
- 사용자가 AI와 대화를 시작할 수 있다
- 세션이 데이터베이스에 저장된다

### 연관 요구사항
- GAME-001, GAME-006

## UC-006: 대화 기반 스토리 진행

### 개요
| 항목 | 내용 |
|------|------|
| **유스케이스명** | 대화 기반 스토리 진행 |
| **ID** | UC-006 |
| **주요 액터** | 로그인 사용자 |
| **설명** | 사용자가 입력을 전송하고 AI가 실시간으로 응답을 생성한다 (SSE 스트리밍) |

### 선행 조건
- 게임 세션이 시작되어 있어야 한다
- API 키가 등록되어 있어야 한다

### 메인 흐름 (Main Flow)
1. 사용자가 입력 상자에 메시지를 입력한다:
   - 선택적 모드 접두사: [행동], [생각], [대사], [장면 지시]
2. 사용자가 "전송"을 클릭한다
3. 프론트엔드는 `POST /api/game/chat`을 호출한다:
   - sessionId
   - userMessage
4. 백엔드는 다음을 수행한다:
   - `sessions` 테이블에서 세션을 조회한다
   - `session_memory` 테이블에서 메모리를 조회한다
   - 시스템 프롬프트 = 스토리 프롬프트 + 메모리 프롬프트
   - 슬라이딩 윈도우 적용 (최근 20개 메시지)
5. 백엔드는 다음을 반환한다:
   - systemPrompt
   - contents (메시지 배열)
   - safetySettings
   - model
6. 프론트엔드는 Gemini API에 직접 SSE 요청을 보낸다:
   - `POST https://generativelanguage.googleapis.com/v1beta/models/{model}:streamGenerateContent`
   - Header: `x-goog-api-key: {사용자 API 키}`
7. Gemini가 응답을 SSE 스트림으로 반환한다
8. 프론트엔드는 실시간으로 텍스트를 렌더링한다
9. 응답 완료 후:
   - 프론트엔드는 상태창 블록(```status...```)을 파싱한다
   - 프론트엔드는 메시지를 `sessions.messages`에 자동 저장한다

### 대안 흐름 (Alternative Flows)
- **9a. 상태창 파싱:**
  - 시스템은 ```status
    HP: 50/100
    MP: 30/100
    ```
    블록을 파싱하여 InfoPanel에 표시한다

### 사후 조건
- 사용자가 AI 응답을 확인하고 대화를 계속할 수 있다
- 세션이 자동 저장된다

### 연관 요구사항
- GAME-002 ~ GAME-006, GAME-009, GAME-010

### 선행 조건
- 사용자가 로그인 상태여야 한다
- 최소 하나 이상의 세션이 존재해야 한다

### 메인 흐름 (Main Flow)

5. 시스템은 세션을 로드한다:
   - `sessions` 테이블에서 세션 조회
   - `session_memory` 테이블에서 메모리 조회
6. 시스템은 대화 기록을 표시한다

### 대안 흐름 (Alternative Flows)
- **2a. 세션 패널:**
  - 플레이 페이지 좌측 패널에서 전체 세션 목록을 표시한다 (최대 50개)

### 사후 조건
- 사용자가 중단된 지점부터 게임을 계속할 수 있다

### 연관 요구사항
- STORY-BROWSE-006, GAME-007

## UC-008: 스토리 생성

### 개요
| 항목 | 내용 |
|------|------|
| **유스케이스명** | 스토리 생성 |
| **ID** | UC-008 |
| **주요 액터** | 스토리 작성자 |
| **설명** | 사용자가 새로운 스토리를 작성하고 저장한다 |

### 선행 조건
- 사용자가 로그인 상태여야 한다

### 메인 흐름 (Main Flow)
1. 사용자가 에디터 페이지(`/editor`)에 접근한다
2. 시스템은 에디터 인터페이스를 표시한다:
   - 왼쪽: 사이드바 (섹션 네비게이션)
   - 중앙: 텍스트 입력 영역
   - 오른쪽: 미리보기 패널
3. 사용자가 다음 정보를 입력한다:
   - **기본 설정**: 제목, 설명, 아이콘, 배너, 장르 태그
   - **세계관**: 배경 설정
   - **스토리**: 개요 및 줄거리
   - **캐릭터**: 등장인물 정보
   - **시스템 규칙**: 게임 룰
   - **출력 설정**: 서술 분량, LaTeX 사용
   - **상태창**: 장르별 프리셋 선택
4. 사용자는 "프롬프트 미리보기"를 클릭하여 조합된 프롬프트를 확인한다

   - is_public = false (기본)
