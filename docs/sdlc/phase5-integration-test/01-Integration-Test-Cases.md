## 1. 개요 (Overview)

### 1.1 테스트 목적

본 문서는 AI Story Game 플랫폼의 통합 테스트 케이스를 정의한다. 통합 테스트는 프론트엔드, 백엔드, 데이터베이스 간의 데이터 흐름과 상호작용을 검증한다.

### 1.2 테스트 범위

| 범위 | 설명 | 관련 유스케이스 |
|------|------|---------------|
| **인증 통합** | Supabase Auth 인증 플로우 | UC-002, UC-003, UC-004 |
| **스토리 관리** | 스토리 CRUD 및 공개 설정 | UC-001, UC-008, UC-009, UC-011 |
| **게임/세션** | 세션 생성, SSE 스트리밍, 메모리 관리 | UC-005, UC-006, UC-007 |
| **관리자 기능** | 대시보드, 로그, 설정, 프리셋 | UC-012, UC-013, UC-014, UC-015 |

### 1.3 테스트 환경

| 항목 | 정보 |
|------|------|
| **프론트엔드** | http://localhost:5173 |
| **백엔드** | http://localhost:3000 |
| **데이터베이스** | Supabase (PostgreSQL) |
| **테스트 도구** | Playwright (E2E + API Testing) |
