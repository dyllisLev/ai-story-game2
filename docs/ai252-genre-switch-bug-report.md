**발견자:** QA Engineer
**우선순위:** P0 (Critical)

## 버그 요약

스토리 에디터에서 장르 프리셋과 제목을 변경하여 새로운 스토리를 생성하고 저장한 후, "게임 시작" 버튼을 클릭하면 이전 세션의 스토리가 로드됩니다. 시스템 프롬프트는 새 장르로 변경되었으나, 실제 게임 콘텐츠는 이전 장르의 스토리가 유지되어 부정합이 발생합니다.

---

## 재현 단계

1. https://aistorygame.nuc.hmini.me/admin 에 접속
2. 기존 스토리(예: 판타지 F1-Basic)에서 에디터 진입
3. 장르 버튼 클릭 (예: 판타지 → 무협)
4. 프리셋 변경 (예: 🐉 에테르나의 전설 → ⚔️ 고전 무협: 마종의 제자)
5. 제목 변경 (예: F1-Basic - 마법학교 입학식 → M1-Basic - 마종의 제자)
6. "저장" 버튼 클릭
7. "게임 시작" 버튼 클릭

**기대 결과:** 새로운 무협 스토리가 시작됨

**실제 결과:** 판타지 스토리가 다시 로드됨

## 영향 분석

### AI-252 태스크 영향
- 9개 장르별 품질 평가가 불가능해짐
- 장르별로 다른 스토리를 생성하여 테스트할 수 없음
- 태스크 차단 (Blocker)

### 프로덕션 환경 영향
- 사용자가 새 장르의 스토리를 생성할 수 없음
- 같은 장르의 스토리만 반복 생성됨
- 사용자 경험 저조 및 기능 오작동
- 비즈니스 영향: 사용자 이탈 가능성

---

## 기술적 원인 (추정)

### 가능한 원인 1: 세션 ID 재사용
- 게임 시작 시 세션 ID를 새로 생성하지 않고 이전 세션을 재사용
- 세션 키가 스토리 ID가 아닌 다른 값(사용자 ID 등)으로 생성되어 같은 사용자의 모든 게임이 같은 세션을 사용

### 가능한 원인 2: 프론트엔드 상태 관리 오류
- 프론트엔드에서 스토리 상태를 변경하지 않고 게임 시작
- React 상태가 에디터 변경을 반영하지 못함

### 가능한 원인 3: 백엔드 API 라우팅 오류
- POST /api/game/start API에서 스토리 ID를 받지 않고 세션만 새로 생성
- 또는 세션 생성 시 스토리 설정을 제대로 전달하지 않음

---

## 권장 수정 방안

### 즉시 조치 (Hotfix)

1. **게임 시작 시 세션 무조건 새로 생성**
   - POST /api/game/start 호출 시 매번 새 세션 ID 생성
   - 이전 세션과 완전히 분리

2. **스토리 변경 시 세션 초기화**
   - 에디터에서 "저장" 후 "게임 시작" 시 이전 세션 삭제
   - 새 스토리 설정으로 새 세션 생성

### 근본적 조치 (Root Cause Fix)

1. **API 검증 강화**
   - POST /api/game/start에서 스토리 ID 필수화
   - 스토리 ID와 현재 세션의 불일치 시 새 세션 생성

2. **프론트엔드 상태 관리 개선**
   - 스토리 에디터 변경 시 게임 세션 상태 초기화 로직 추가
   - useEffect 또는 적절한 생명주기 훅으로 상태 동기화

3. **테스트 케이스 추가**
   - 장르 변경 후 게임 시작 E2E 테스트
   - 다중 스토리 생성 테스트
